Գլխավոր Html-ի տեղեկատու <html>

<html>

Элемент <html> является контейнером, который заключает в себе всё содержимое веб-страницы, включая элементы <head> и <body>. Открывающий и закрывающий теги <html> в документе не обязательны, но хороший стиль диктует непременное их использование. Как правило, <html> идёт в документе вторым, после определения типа документа (Document Type Definition, DTD), устанавливаемого через <!DOCTYPE>. Закрывающий тег <html> всегда стоит в документе последним.

Օրինակ 

<html>
  ...
</html>

Փակող թեգ

Պարտադիր չէ:

Հատկանիշներ

manifest
Указывает файл манифеста, необходимый для создания оффлайнового приложения.
version
Задаёт версию DTD. Данный атрибут не нужен и избыточен, версия определяется через <!DOCTYPE>.
xmlns
Указывает пространство имён для XHTML-документов.

Также для этого элемента доступны универсальные атрибуты.

Օրինակ

<!DOCTYPE HTML> <html> <head> <!-- Этот раздел предназначен для заголовка страницы и технической информации. --> </head> <body> <!-- А здесь надо размещать все, что хочется увидеть на странице. --> </body> </html>

Հստակեցում

Յուրաքանչյուր հստակեցում անցնում է մի քանի փուլեր հաստատման համար։

  • Recommendation (Рекомендация) — Հստակեցումը հաստատվել է W3C-ով, եւ խորհուրդ է տրվում որպես ստանդարտ:
  • Candidate Recommendation (Возможная рекомендация) — Ստանդարտի համար պատասխանատու խումբը բավարարված է, քանի որ այն համապատասխանում է իր նպատակներին, սակայն պահանջվում է մշակողների խմբի աջակցությունը ստանդարտի իրականացման համար:
  • Proposed Recommendation (Предлагаемая рекомендация) — Այս փուլում փաստաթուղթը ներկայացվել է W3C խորհրդատվական խորհրդի քննարկմանը ՝ վերջնական հաստատման համար ։
  • Working Draft (Рабочий проект) —Նախագծի ավելի հասուն տարբերակը քննարկումից հետո եւ ուղղումներ ներդրումը որպեսզի խումբը դիտարկի։
  • Editor's draft (Редакторский черновик) — Նախագծի խմբագիրների կողմից խմբագրումից հետո ստանդարտի նախագծի նախագիծ:
  • Draft (Черновик спецификации) — Ստանդարտի առաջին նախագիծը:

Առանձնատունը HTML-ի (Living) — կենդանի ստանդարտն է, այն չի պահպանում տարբերակների ավանդական համարակալումը, քանի որ գտնվում է մշտական մշակման մեջ եւ պարբերաբար թարմացվում է:

  •  — Տարրը ամբողջությամբ ապահովվում է զննարկչի կողմից;
  •  — Տարրը չի ընկալվում զննարկչի կողմից և անտեսվում է;
  •  — Գործողության ընթացքում կարող են հայտնվել տարբեր սխալներ, կամ էլեմենտը աջակցվում է նախազգուշացումներով;
